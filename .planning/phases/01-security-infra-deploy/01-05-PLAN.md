<task type="checkpoint:human-action">
  <name>Task 2: GitHub branch protection 활성화</name>
  <what-built>
    .github/workflows/ci.yml이 생성되어 PR에서 lint-typecheck, build, unit-test, e2e 4종 job이 자동 실행됩니다 (per D-13, D-14, D-15).
    GitHub branch protection을 설정하면 4종 체크 중 하나라도 실패 시 PR merge 버튼이 비활성화됩니다.
  </what-built>
  <how-to-verify>
    1. GitHub → danjiondo repository → Settings 탭
    2. Code and automation → Branches
    3. "Add branch protection rule" 클릭
    4. Branch name pattern: `main`
    5. 다음 옵션 활성화:
       - [x] Require status checks to pass before merging
       - [x] Require branches to be up to date before merging
    6. "Status checks that are required" 검색창에서 다음 4개 추가:
       - `Lint & Typecheck`
       - `Build`
       - `Unit Tests (Vitest)`
       - `E2E Tests (Playwright)`

       (이 이름들은 ci.yml의 `name:` 필드와 정확히 일치해야 함)

    7. 선택사항 (권장):
       - [x] Require pull request before merging
       - [x] Dismiss stale pull request approvals when new commits are pushed

    8. "Create" 또는 "Save changes" 클릭

    확인:
    - test PR을 생성하거나 기존 PR에서 Check 탭에서 4종 job이 실행되는지 확인
    - 모든 체크 통과 시에만 "Merge pull request" 버튼 활성화됨
  </how-to-verify>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | Vercel 환경변수 (SUPABASE_SERVICE_ROLE_KEY) | mitigate | Vercel Secrets: "Production only" 범위로 설정. Preview/Development 환경에 service_role 노출 금지 |
| T-05-02 | Tampering | GitHub branch protection 미설정 | mitigate | 이 플랜의 Task 2에서 필수 설정. 설정 전까지 PR merge에 품질 게이트 없음 |
