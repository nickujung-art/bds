```typescript
function CafeLink({ complexName }: { complexName: string }) {
  const href = `https://cafe.naver.com/ArticleSearchList.nhn?search.query=${encodeURIComponent(complexName)}`
  return (
    <a
      href={href}
      target="_blank"
      rel="noopener noreferrer"
      aria-label={`네이버 카페에서 ${complexName} 이웃 글 보기 (새 탭에서 열림)`}
      style={{
        display:        'inline-flex',
        alignItems:     'center',
        gap:            4,
        padding:        '8px 0 12px',
        font:           '500 11px/1 var(--font-sans)',
        color:          'var(--fg-brand)',
        textDecoration: 'none',
        borderBottom:   '1px solid var(--line-subtle)',
        marginBottom:   12,
        width:          '100%',
      }}
    >
      네이버 카페에서 {complexName} 이웃 글 보기 ↗
    </a>
  )
}
```

**ReviewList 컴포넌트에서 complexName prop 수신:**
ReviewList의 props에 `complexName?: string`이 없다면 추가한다. 단지 상세 page.tsx에서 복합명을 전달해야 한다. ReviewList가 이미 complexId를 받고 있으면, page.tsx에서 complexName도 함께 전달하도록 수정한다.

CafeLink는 후기 목록 컨테이너 최상단에 위치:
```typescript
return (
  <div>
    {complexName && <CafeLink complexName={complexName} />}
    {/* 기존 후기 목록 */}
    ...
  </div>
)
```

**hover 스타일:**
CSS-in-JS는 hover를 직접 처리하기 어려우므로 `onMouseEnter`/`onMouseLeave`로 처리하거나, 기존 코드베이스에 hover 처리 패턴이 있으면 그 방식을 따른다. 없으면 생략하고 UI-SPEC의 hover underline은 Phase 5에서 추가.
