**getComplexesForMap 쿼리 수정:**

SELECT에 `si, gu, dong` 추가. recent_price/date/area_m2는 transactions 테이블에서 최근 1건 가져오는 방식:

Supabase JS client에서 transactions LATERAL JOIN이 불가하므로, complexes SELECT 후 각 단지별 최근 거래를 가져오는 대신, 두 가지 방법:

방법 A (권장): complexes에 `recent_price`, `recent_deal_date`, `recent_area_m2` 컬럼을 직접 추가하는 마이그레이션 없이, getComplexesForMap에서 **별도 IN 쿼리**로 complex_ids의 최근 거래를 한 번에 가져와 Map으로 조합.

실제 구현:
1. complexes 쿼리로 기존 필드 + si/gu/dong 가져오기
2. 얻은 ids를 사용해 transactions 쿼리: `SELECT complex_id, price, deal_date, area_m2 WHERE complex_id IN (...) AND deal_type='sale' AND cancel_date IS NULL AND superseded_by IS NULL ORDER BY deal_date DESC` — 이를 각 complex_id별 최초 행만 취함
3. Map<complex_id, {price, date, area}> 구성 후 ComplexMapItem에 merge

```typescript
// 스텝 2: 최근 거래 1건씩 조회 (CRITICAL: cancel_date IS NULL AND superseded_by IS NULL 필수)
const { data: txData } = await supabase
  .from('transactions')
  .select('complex_id, price, deal_date, area_m2')
  .in('complex_id', ids)
  .eq('deal_type', 'sale')
  .is('cancel_date', null)
  .is('superseded_by', null)
  .order('deal_date', { ascending: false })

// Map 구성: 각 complex_id의 첫 번째(최신) 거래만 유지
const recentTxMap = new Map<string, { price: number; deal_date: string; area_m2: number }>()
for (const tx of (txData ?? [])) {
  if (!recentTxMap.has(tx.complex_id)) {
    recentTxMap.set(tx.complex_id, {
      price:     tx.price,
      deal_date: tx.deal_date,
      area_m2:   tx.area_m2,
    })
  }
}
```

단, supabase .in()은 최대 1000개 처리 가능. complexes 수가 669개이므로 안전.
