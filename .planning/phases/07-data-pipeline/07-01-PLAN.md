**buildDongMap 함수:** fetchComplexList를 sgg_code별로 1회 호출하여 kaptCode → as3(dong) 매핑 Map 생성.

**main 함수 로직:**

1. regions 테이블에서 sgg_code, si, gu 조회 → regionMap (Map<string, {si, gu}>) 생성
2. buildDongMap(sggCodes) 호출 (6번 fetchComplexList API 호출)
3. complexes에서 WHERE kapt_code IS NOT NULL AND si IS NULL 조회 (id, kapt_code, sgg_code 컬럼)
4. 각 complex에 대해:
   a. fetchKaptBasicInfo(complex.kapt_code) 호출
   b. si/gu = regionMap.get(complex.sgg_code) ?? { si: null, gu: null }
   c. dong = dongMap.get(complex.kapt_code) ?? null
   d. built_year = kaptUsedate ? parseInt(kaptUsedate.slice(0, 4), 10) : null
   e. heat_type = info.heatType ?? info.codeHeatNm ?? null
   f. **JSONB merge (다른 키 보존):**
      ```typescript
      // data_completeness는 JSONB merge — 전체 덮어쓰기 금지 (07-02가 병렬로 transactions 키 설정)
      // Postgres JSONB merge: COALESCE(data_completeness, '{}') || '{"kapt":true}'
      // supabase-js에서는 raw SQL update 사용:
      await supabase.rpc('jsonb_set_key', {
        p_table: 'complexes',
        p_id: complex.id,
        p_key: 'kapt',
        p_value: true,
      })
      // 만약 jsonb_set_key RPC가 없다면 아래 패턴 사용:
      // supabase.from('complexes')
      //   .update({ si, gu, dong, road_address: doroJuso, ... })
      //   .eq('id', complex.id)
      // 그 후 data_completeness는 별도 raw query로 merge:
      // UPDATE complexes SET data_completeness = COALESCE(data_completeness,'{}') || '{"kapt":true}' WHERE id = $1
      // 가장 단순한 구현: supabase.rpc('merge_data_completeness_kapt', { p_id: complex.id })
      //   — 없으면 두 개의 update 호출 (첫 번째는 나머지 필드, 두 번째는 data_completeness merge)
      ```
      **실제 구현 지침:** kapt-enrich.ts에서는 두 단계 update로 처리한다:
      - 1단계: si, gu, dong, road_address, household_count, built_year, heat_type update
      - 2단계: `supabase.from('complexes').update({ data_completeness: supabase.rpc(...) })` 패턴이 지원 안 되므로, raw SQL로 merge:
        ```typescript
        await supabase.rpc('exec_sql', { sql: `UPDATE complexes SET data_completeness = COALESCE(data_completeness,'{}') || '{"kapt":true}' WHERE id = '${complex.id}'` })
        ```
        OR 더 안전하게: 기존 data_completeness를 먼저 읽어서 JavaScript에서 merge 후 전체 update:
        ```typescript
        const existing = complex.data_completeness as Record<string, boolean> ?? {}
        const merged = { ...existing, kapt: true }
        // 이미 조회한 데이터에 kapt: true만 추가하여 다른 키 보존
        await supabase.from('complexes')
          .update({ si, gu, dong, road_address: doroJuso, household_count: kaptdaCnt,
                    built_year, heat_type, data_completeness: merged })
          .eq('id', complex.id)
        ```
        **이 방식을 사용할 것.** complexes SELECT 쿼리에 data_completeness 컬럼 포함하여 조회.
   g. 100ms 대기 (KAPT rate limit 방어)
5. 진행 상황 출력: `[N/669] 단지명 ... done`
6. 완료 요약: 성공/실패 카운트
